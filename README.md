# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - обновляет `<img>`.  
`setText(...)`, `setDisabled(...)`, `toggleClass(...)`, `setChildren(...)` - короткие DOM-утилиты для наследников.


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`off(event: EventName, callback: Subscriber): void` - отписка от события; удаляет обработчик, а при отсутствии подписчиков удаляет событие из коллекции.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`onAll(callback: (event: EmitterEvent) => void): void` - подписка на все события через специальное событие `*`.  
`offAll(): void` - очищает все подписки брокера событий.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


### Данные

В приложении используются две основные сущности: товар и покупатель. Их структура описана интерфейсами.

#### Интерфейс IProduct (Продукт)
```ts
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}
```

`IProduct` описывает данные товара, которые приходят с сервера и используются для отображения карточек и списка каталога.  

`id` — айди товара.  
`description` — текстовое описание товара.  
`image` — ссылка на изображение товара.  
`title` — название товара.  
`category` — категория товара.  
`price` — цена товара.

#### Интерфейс IBuyer (Покупатель)
```ts
interface IBuyer {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}
```
`IBuyer` описывает данные покупателя, которые используются при оформлении заказа.  
`payment` — выбранный способ оплаты (тип `TPayment`).  
`email` — адрес электронной почты.  
`phone` — контактный телефон.  
`address` — адрес доставки.

`TPayment` — тип, описывающий доступные способы оплаты.

### Модели данных

#### Класс ProductsCatalog (каталог товаров)
Хранит полный список товаров и выбранный товар для подробного отображения. Отвечает за доступ к данным каталога и поиск товара по идентификатору.

Конструктор:  
`constructor(events?: IEvents)` - инициализирует пустой каталог товаров (`items`) и отсутствие выбранного товара (`preview = null`). При передаче брокера событий эмитит события при изменении данных.

Поля класса:  
`items: IProduct[]` — массив всех товаров, доступных в каталоге.  
`preview: IProduct | null` — товар, выбранный для подробного отображения; `null`, если выбор отсутствует.

Методы класса:  
`setItems(items: IProduct[]): void` — сохраняет полученный массив товаров в модели каталога и эмитит событие `catalog:changed`.  
`getItems(): IProduct[]` — возвращает копию массива товаров.  
`getItem(id: string): IProduct | undefined` — возвращает товар по его `id`, либо `undefined`, если товар не найден.  
`setPreview(item: IProduct | null): void` — сохраняет товар для подробного отображения или очищает выбор и эмитит событие `catalog:previewChanged`.  
`getPreview(): IProduct | null` — возвращает выбранный товар для подробного отображения.

#### Класс Basket (корзина)
Хранит товары, выбранные пользователем для покупки. Отвечает за операции добавления, удаления и вычисления итогов.

Конструктор:  
`constructor(events?: IEvents)` - инициализирует пустой массив товаров корзины. При передаче брокера событий эмитит событие `basket:changed` при изменении состава корзины.

Поля класса:  
`items: IProduct[]` — массив товаров, находящихся в корзине.

Методы класса:  
`getItems(): IProduct[]` — возвращает массив товаров в корзине.  
`addItem(item: IProduct): void` — добавляет переданный товар в корзину и эмитит событие `basket:changed`.  
`removeItem(item: IProduct): void` — удаляет переданный товар из корзины и эмитит событие `basket:changed`, если удаление произошло.  
`clear(): void` — очищает корзину и эмитит событие `basket:changed`, если корзина не была пустой.  
`getTotal(): number` — возвращает суммарную стоимость товаров (позиции с `price = null` не учитываются).  
`getCount(): number` — возвращает количество товаров в корзине.  
`hasItem(id: string): boolean` — проверяет наличие товара в корзине по его `id`.

#### Класс Buyer (покупатель)
Хранит данные покупателя, необходимые для оформления заказа, и отвечает за их валидацию.

Конструктор:  
`constructor(events?: IEvents)` — инициализирует пустые значения полей покупателя. При передаче брокера событий эмитит событие `buyer:changed` при изменении данных.

Поля класса:  
`payment: TPayment | null` — выбранный способ оплаты; `null`, если способ оплаты не выбран.  
`email: string` — адрес электронной почты.  
`phone: string` — контактный телефон.  
`address: string` — адрес доставки.

Методы класса:  
`setData(data: Partial<IBuyer>): void` — сохраняет переданные значения полей, не затрагивая остальные, и эмитит событие `buyer:changed`, если данные изменились.  
`getData(): Partial<IBuyer>` — возвращает текущие данные покупателя; поле `payment` присутствует только если способ оплаты выбран.  
`clear(): void` — очищает все данные покупателя и эмитит событие `buyer:changed`, если до очистки в модели были данные.  
`validate(): Partial<Record<keyof IBuyer, string>>` — проверяет, что поля не пустые, и возвращает объект с текстами ошибок по каждому неверному полю.


### Слой представления (View)

Компоненты слоя представления отвечают только за работу с DOM, отображение данных и уведомление Презентера о действиях пользователя. Все классы наследуются от `Component<T>`, не хранят бизнес-данные и получают все нужные DOM-элементы в конструкторе. `render()` без аргументов возвращает корневой DOM-элемент компонента.

### События приложения

События генерируются как View-компонентами, так и моделями данных. Все они обрабатываются Презентером.

`catalog:changed` — в модели каталога обновлён список товаров; в событие передаётся `items`.  
`catalog:previewChanged` — в модели каталога изменён товар предпросмотра; в событие передаётся `preview`.  
`basket:changed` — в модели корзины изменился состав товаров; в событие передаются `items`, `total` и `count`.  
`buyer:changed` — в модели покупателя изменились данные; в событие передаются `data` и `errors`.  

`basket:open` — пользователь нажал кнопку корзины в шапке.  
`card:select` — пользователь нажал карточку товара в каталоге; в событие передаётся `id` товара.  
`card:add` — пользователь нажал кнопку покупки в предпросмотре; в событие передаётся `id`.  
`card:remove` — пользователь нажал кнопку удаления из корзины в предпросмотре; в событие передаётся `id`.  
`basket:remove` — пользователь нажал кнопку удаления товара в корзине; в событие передаётся `id` товара.  
`basket:submit` — пользователь нажал кнопку перехода к оформлению заказа в корзине.  
`order.payment:change` — пользователь выбрал способ оплаты; в событие передаётся `payment`.  
`order.address:change` — пользователь изменил адрес доставки; в событие передаются `field` и `value`.  
`order:submit` — пользователь отправил форму заказа.  
`contacts.email:change` — пользователь изменил e-mail; в событие передаются `field` и `value`.  
`contacts.phone:change` — пользователь изменил телефон; в событие передаются `field` и `value`.  
`contacts:submit` — пользователь отправил форму контактов.  
`modal:close` — пользователь запросил закрытие модального окна (кнопка, оверлей или `Escape`).  
`success:close` — пользователь нажал кнопку закрытия экрана успешного заказа.

### Презентер

Презентер реализован в основном скрипте `src/main.ts`, без выделения отдельного класса. 

`main.ts` создаёт экземпляры всех моделей, представлений, API-классов и общего брокера событий.  
`main.ts` подписывается на все события моделей и представлений.  
`main.ts` изменяет данные только через методы моделей и обновляет отображение только при обработке событий моделей или при открытии модального окна.

Основной поток работы:
после загрузки товаров с сервера Презентер сохраняет их в `ProductsCatalog`, а затем по событию `catalog:changed` рендерит карточки каталога;  
по событиям пользовательских действий Презентер изменяет состояние моделей (`ProductsCatalog`, `Basket`, `Buyer`);  
по событиям моделей Презентер синхронизирует интерфейс: перерисовывает каталог, счётчик корзины, содержимое модального окна и формы;  
при отправке второй формы Презентер собирает заказ, отправляет его через `WebLarekApi`, а после успешного ответа показывает экран успешного оформления и очищает модели корзины и покупателя.



## Слой коммуникации

### Класс WebLarekApi
`WebLarekApi` — класс коммуникационного слоя, который работает через композицию с объектом, реализующим интерфейс `IApi`.

Конструктор:  
`constructor(api: IApi)` — принимает зависимость для HTTP-запросов (объект, реализующий `IApi`) и сохраняет её в поле класса.

Методы:
`getProducts(): Promise<IProduct[]>` — выполняет `GET /product/`, получает объект `IProductsResponse` и возвращает массив товаров `items`.

`createOrder(order: IOrder): Promise<IOrderResult>` — выполняет `POST /order/` и отправляет данные заказа в формате `IOrder`:
- данные покупателя (`IBuyer`),
- список id товаров (`items: string[]`),
- итоговую сумму (`total: number`).

Ответ сервера при оформлении заказа описан типом `IOrderResult` (`id` и `total`).
